Tips
----
• For a single public IP on TCP+UDP, keep both enabled with same type+port; chart will emit one LoadBalancer service with both ports.
• If you ever change the ports list and Kubernetes rejects a patch due to port order, delete the affected Service and re-upgrade.
• In each remote device, set Addresses like: tcp://your.domain:22000,quic://your.domain:22000
• Global announce & relays are disabled by default in this chart; LAN announce is on.
